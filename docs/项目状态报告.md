## 1. 项目概述

Claudio 是一个**个人 AI 电台主持人**应用，通过 Express + WebSocket 双通道架构，将 LLM 对话与音乐播放深度融合。用户可与 AI 主持人闲聊、点歌、获取场景化音乐推荐，系统亦支持定时自动唤醒（早安/午间/晚间问候）。

---

## 2. 技术栈总览

| 层级 | 技术选择 | 说明 |
|------|---------|------|
| **运行时** | Node.js >= 20, ESM | 纯 ES Module 项目 |
| **Web 框架** | Express ^4.21 | REST API |
| **实时通信** | ws ^8.18 | WebSocket 双向推送 (/stream) |
| **LLM 适配** | OpenAI SDK ^6.38 + Claude Code CLI + Ollama HTTP | 4 provider 多级降级 (claude → openai → ollama → mock) |
| **音乐** | NeteaseCloudMusicApi ^4.32 | 网易云搜索/播放/歌词/推荐 |
| **定时任务** | node-cron ^4.2 | 3 条 cron 定时触发 |
| **存储** | SQLite (node:sqlite DatabaseSync) | WAL 模式, 5 张业务表 |
| **TTS** | Fish Audio API + 浏览器 Web Speech | 双通道兜底 |
| **向量检索** | OpenAI 兼容 embeddings API + SQLite | RAG 语义记忆 |
| **前端** | 零框架 Vanilla JS | 约 750 行 app.js, Service Worker |
| **进程管理** | 无 (单进程) | 建议 PM2 生产化 |

## 4. 服务端 API 完整清单

### 4.1 REST API (29 个端点)

#### 核心对话
| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/chat` | 主聊天入口，消息分流 → LLM → 解析 → 持久化 → 广播 |
| GET | `/api/messages` | 分页获取历史消息 (支持 `?limit=` & `?before=`) |

#### 状态管理
| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/now` | 当前播放歌曲 + 状态信息 |
| GET | `/api/health` | 健康检查 (LLM 状态/场景/品味/缓存) |
| POST | `/api/state/clear` | 清空全部对话历史 |
| POST | `/api/state/prune` | 按天数或条数裁剪历史 |
| POST | `/api/reload` | 热重载 persona / memory / schedule |

#### 音乐
| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/music/search` | 搜索歌曲 (q → 搜索结果) |
| GET | `/api/music/play` | 播放 / 点歌 (keyword → 解析 → 直链) |
| GET | `/api/music/lyrics` | 获取歌词 (songId → 时间轴) |
| GET | `/api/music/recommend` | 相似推荐 (songId → 相似歌曲) |
| GET | `/api/music/refresh` | 刷新过期直链 (过期 URL 重新解析) |

#### UPnP/DLNA
| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/upnp/scan` | 扫描局域网 DLNA 设备 (SSDP) |
| POST | `/api/upnp/push` | 推送当前歌曲到指定设备 |
| POST | `/api/upnp/play` | 控制 DLNA 设备播放 |

#### Radio
| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/radio/start` | 启动 Radio 自动播放模式 |
| POST | `/api/radio/stop` | 停止 Radio |
| GET | `/api/radio/next` | Radio 获取下一首 |

#### 配置与上下文
| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/context` | 返回当前完整上下文 (天气/日程/品味/场景) |
| GET | `/api/context/mood` | 心情记录 CR (GET 列表 / POST 新增) |
| GET | `/api/rss` | RSS 资讯摘要 |
| GET | `/api/settings` | 获取 LLM 设置 (provider/model) |
| POST | `/api/settings` | 更新 LLM 设置 (provider/model) |
| GET | `/api/profile/read` | 读取品味文件 (taste/routines/playlists/mood-notes) |
| POST | `/api/profile/save` | 保存品味文件 |
| GET | `/api/schedule` | 定时任务列表 |
| POST | `/api/schedule` | 添加/更新定时任务 |
| POST | `/api/rag/index` | 触发 RAG 重新索引 |

### 4.2 WebSocket 消息类型 (`/stream`)

**客户端 → 服务端**：
| type | 描述 |
|------|------|
| `ping` | 心跳 (服务端回复 pong) |

**服务端 → 客户端**：
| type | 描述 |
|------|------|
| `hello` | 连接确认 |
| `user-echo` | 用户消息回显 |
| `thinking` | LLM 思考中 |
| `resolving` | 正在搜索歌曲 |
| `say` | LLM 回复 (含 text/play/ttsUrl/segue) |
| `now-playing` | 歌曲已解析，推送播放 |
| `now-playing-failed` | 点歌失败 |
| `radio-started` | Radio 模式启动 |
| `radio-stopped` | Radio 模式停止 |
| `radio-track` | Radio 下一首 |
| `radio-track-failed` | Radio 选歌失败 |
| `history-cleared` | 对话历史已清空 |
| `error` | 错误信息 |

| **导航** | 三页路由 (Player / Profile / Settings), hash-based SPA |

## 流式输出调研（Day 11）

**结论：暂不实现。**

| 维度 | 分析 |
|------|------|
| 当前架构 | LLM 返回完整 JSON（`say`+`play`+`segue`），前端一次性渲染，逻辑简单可靠 |
| 流式收益 | `say` 文本可逐字展示，体验略好；但 `play`/`segue` 需完整解析后才能触发点歌 |
| 流式成本 | OpenAI `stream` 模式需逐 chunk 拼接再解析 JSON，中途解析失败率高；Claude CLI 本身不支持 SSE 流式输出 |
| 前端改动 | 需将 `say` 从一次性渲染改为增量追加，涉及多处状态管理 |
| 结论 | 收益有限、改动面广、Claude 适配器无法支持。**推迟到第 3 周以后考虑。** |
